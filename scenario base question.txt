3.What is new in Dotnet?
1. Unified & Cross-Platform
	.NET is now a single unified platform for Web, APIs, Desktop, Mobile and Cloud.
	Same runtime and libraries across Windows, Linux, and macOS.

2. Performance Improvements
	Every release focuses on runtime performance.
	Faster startup, lower memory usage, better garbage collection and async performance.

3. Minimal APIs
	Introduced Minimal APIs for building lightweight services.
	Less boilerplate compared to traditional controllers.
	Very useful for microservices and serverless APIs.

4. Native AOT (Ahead-of-Time Compilation)
	Supports compiling apps into native binaries.
	Results in very fast startup and smaller memory footprint.
	Ideal for containers and cloud workloads.

5. Cloud & Microservices First
	Built-in support for Docker and Kubernetes.
	Better logging, health checks, and observability.
	Designed to be cloud-native and scalable.

6. Continuous C# Language Enhancements
	Modern C# features like records, pattern matching, init properties, required members.
	Improves code readability, safety, and productivity.

7. MAUI (for Mobile & Desktop)
	.NET now includes MAUI (Multi-platform App UI).
	Used to build Android, iOS, Windows, and macOS apps from a single C# codebase.

Multiple users updating same data (Concurrency)

	- In EF Core, concurrency is handled using optimistic concurrency.
	If we don’t use RowVersion or Timestamp, EF cannot detect concurrent updates and the last user overwrites data.
	By adding a RowVersion column, EF includes it in the WHERE clause during update.
	- If another user has already modified the record, EF throws DbUpdateConcurrencyException.
	When DbUpdateConcurrencyException occurs, we handle it based on business needs.
	Usually we inform the user and reload latest data.
	In some cases, we retry or merge changes instead of overwriting.
