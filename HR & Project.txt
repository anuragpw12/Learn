Why microservice?
	We initially had a monolithic system, but since there were frequent client-specific changes, 
	every modification required redeploying the entire application. To reduce deployment impact 
	and improve maintainability, we separated some modules into independent services so they could 
	be deployed and updated independently.

DB?
	Since the system originally started as a monolithic application and the team size is relatively small,
	maintaining separate databases would add unnecessary operational complexity. Our goal was to modularize 
	the application first while keeping database management simpler.
	
	Since our platform supports multiple clients, having a separate database for every service would create
	significant operational overhead. So we separated services at the application level but kept a shared 
	database with different tables for each module.
