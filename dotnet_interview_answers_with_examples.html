ASP.NET Core (30)

Q1Explain the ASP.NET Core request pipeline
How to answerFrame it as a journey: 'A request enters Kestrel, flows through the middleware pipeline in the exact order I register it, hits the matched endpoint, and the response flows back out through the same middleware in reverse.' Then name the order, because order is what they're testing.
var app = builder.Build();
app.UseExceptionHandler("/error");   // outermost — catches everything below
app.UseHttpsRedirection();
app.UseStaticFiles();
app.UseRouting();                    // selects the endpoint
app.UseAuthentication();             // who are you
app.UseAuthorization();              // what can you do
app.MapControllers();                // endpoint executes
app.Run();
Q2Middleware vs Filters / Action Filters
How to answerDraw the boundary clearly: 'Middleware runs for every request at the HTTP level — before MVC even exists. Filters run inside MVC and can see the action, its arguments and the result.' Then give the use-case split.
// Middleware — runs for ALL requests, even static files
app.Use(async (ctx, next) => { _log.LogInformation(ctx.Request.Path); await next(); });

// Action filter — only for controller actions, sees model + args
public class AuditFilter : IActionFilter {
    public void OnActionExecuting(ActionExecutingContext ctx)
        => _audit.Log(ctx.ActionArguments);   // has MVC context
    public void OnActionExecuted(ActionExecutedContext ctx) { }
}
Q3Dependency Injection & service lifetimes
How to answerDefine the three in one breath, then immediately give the classic bug — interviewers want to know you understand the *consequence* of getting it wrong, not just the definitions.
builder.Services.AddSingleton<ICache, Cache>();      // one for the whole app
builder.Services.AddScoped<AppDbContext>();          // one per HTTP request
builder.Services.AddTransient<IEmailBuilder, EmailBuilder>(); // new every time

// The classic bug — captive dependency:
// Injecting a Scoped DbContext into a Singleton keeps ONE DbContext alive
// forever, shared across all requests → threading errors + stale data.
Q4Why is AddScoped() commonly used?
How to answerOne line: 'Because request-bound services need to be shared within a request but isolated between concurrent requests — and that's exactly what Scoped gives you.' Name the usual suspects.
// DbContext, repositories, unit-of-work, current-user context:
builder.Services.AddScoped<IOrderRepository, OrderRepository>();
// shared across everything in ONE request; a fresh one for the next request.
Q5How does async/await improve scalability?
How to answerCorrect the common misconception up front: 'It doesn't make a single request faster — it improves throughput. During an I/O wait the thread goes back to the pool instead of blocking, so the same threads serve many more concurrent requests.' Then show the anti-pattern.
// BAD — thread blocked for the whole 2s wait, serving nobody
var data = httpClient.GetStringAsync(url).Result;   // also risks deadlock

// GOOD — thread released during the wait, picks up other work
var data = await httpClient.GetStringAsync(url, cancellationToken);

How to answerWalk the flow in one sentence, then make the distinction they're fishing for: 'JWT is a token *format*; OAuth2 is the authorization *framework* that hands them out.'

Q7IActionResult vs ActionResult<T>
How to answer'Both let me return NotFound() or Ok(), but ActionResult<T> also lets me return the object directly and — crucially — tells Swagger the response type.'
public ActionResult<Order> Get(int id) {
    var order = _repo.Find(id);
    if (order is null) return NotFound();  // an action result
    return order;                          // T directly — implicit conversion
}
Q8Global / centralized exception handling
How to answer'I put one handler at the very top of the pipeline, log with a correlation ID, and return RFC 7807 ProblemDetails — never a raw stack trace in production.' Mention IExceptionHandler is the .NET 8 way.
public class GlobalExceptionHandler : IExceptionHandler {
    public async ValueTask<bool> TryHandleAsync(HttpContext ctx, Exception ex, CancellationToken ct) {
        _log.LogError(ex, "Unhandled");
        ctx.Response.StatusCode = ex switch {
            NotFoundException => 404, ValidationException => 400, _ => 500 };
        await ctx.Response.WriteAsJsonAsync(new ProblemDetails {
            Title = "Error", Status = ctx.Response.StatusCode }, ct);
        return true;
    }
}
builder.Services.AddExceptionHandler<GlobalExceptionHandler>();
app.UseExceptionHandler();

Q12How would you implement idempotent APIs?
How to answer'The client sends a unique Idempotency-Key; I store the key with its result and replay that result on retries instead of re-executing. A unique constraint on the business key is the safety net.'
[HttpPost("payments")]
public async Task<IActionResult> Pay(
    [FromHeader(Name="Idempotency-Key")] string key, PaymentDto dto) {
    if (await _store.GetAsync(key) is {} prior) return Ok(prior); // replay
    var result = await _payments.ProcessAsync(dto);
    await _store.SaveAsync(key, result);
    return Ok(result);
}
Q13PUT vs PATCH - when to use?
How to answer'PUT replaces the whole resource and is idempotent; PATCH is a partial update. Send the full object to PUT, only the changed fields to PATCH.'
PUT   /orders/5   { full order object }              // replaces, idempotent
PATCH /orders/5   [{ "op":"replace",
                     "path":"/status",
                     "value":"Shipped" }]             // JSON Patch, partial

Q16How to handle CORS in ASP.NET Core?
How to answer'CORS is a browser rule — the server declares which origins may call it cross-origin. I register a named policy and place UseCors between routing and authorization. Never AllowAnyOrigin with AllowCredentials.'
builder.Services.AddCors(o => o.AddPolicy("spa", p =>
    p.WithOrigins("https://app.example.com").AllowAnyHeader().AllowAnyMethod()));
app.UseRouting();
app.UseCors("spa");
app.UseAuthorization();
Q17Throw vs throw ex - difference?
How to answerShort and definitive: 'Plain throw preserves the original stack trace; throw ex resets it to the current line and you lose where the error actually happened. Always use plain throw.'
catch (Exception ex) {
    _log.LogError(ex, "failed");
    throw;       // preserves original stack trace
    // throw ex; //  resets it — never do this
}

Q28What is IHostedService / BackgroundService?
How to answer'For long-lived background work inside the host — queue consumers, scheduled jobs, cache warmup. BackgroundService gives me an ExecuteAsync loop with a stopping token. For real scheduling I'd reach for Hangfire or Quartz.'
public class QueueWorker : BackgroundService {
    protected override async Task ExecuteAsync(CancellationToken ct) {
        while (!ct.IsCancellationRequested) {
            var msg = await _queue.DequeueAsync(ct);
            await ProcessAsync(msg);
        }
    }
}
builder.Services.AddHostedService<QueueWorker>();
Q29Generic Host vs Web Host - why was Generic Host introduced?
How to answer'Web Host was HTTP-only. The Generic Host generalized DI, config, logging and lifetime for any workload — web apps, worker services, console apps — so since 3.0 web apps run on it with a web workload plugged in.'
Q30How would you design health checks, readiness & liveness probes?
How to answer'Liveness answers is the process alive — fail it and the orchestrator restarts me, so keep it cheap and dependency-free. Readiness answers are my dependencies up — fail it and I'm pulled from the load balancer. I tag dependency checks as ready-only.'
builder.Services.AddHealthChecks()
    .AddSqlServer(conn, tags: new[] { "ready" })
    .AddRedis(redis, tags: new[] { "ready" });

app.MapHealthChecks("/health/live");   // liveness: cheap, no deps
app.MapHealthChecks("/health/ready",
    new() { Predicate = c => c.Tags.Contains("ready") });

Q31Angular lifecycle hooks

ngOnChanges  → ngOnInit → ngDoCheck
  → ngAfterContentInit/Checked → ngAfterViewInit/Checked → ngOnDestroy

ngOnInit()      { this.loadData(); }          // init, not the constructor
ngAfterViewInit(){ this.chart.render(); }      // DOM is ready here
ngOnDestroy()   { this.sub.unsubscribe(); }    // prevent memory leaks
Q32Change detection strategy / how it works
How to answer'Zone.js patches async APIs and triggers change detection from the root. Default checks the whole tree; OnPush only checks a component when an @Input reference changes, an event fires inside it, or an async pipe emits. Signals go further with fine-grained updates.'
@Component({ changeDetection: ChangeDetectionStrategy.OnPush })
export class OrderCard {
  @Input() order!: Order;   // must pass a NEW reference to re-render
}
Q33Observables vs Promises
How to answer'Observables are lazy, cancellable and emit many values over time with operators; Promises are eager, single-value, not cancellable. That's exactly why HttpClient returns Observables — I can cancel or switch requests.'
this.search.valueChanges.pipe(
  debounceTime(300),
  switchMap(term => this.api.search(term))  // cancels the previous request
).subscribe(results => this.results = results);

How to answer'Reactive: the model lives in TypeScript — synchronous, testable, dynamic controls, custom validators — best for complex forms. Template-driven: ngModel in HTML, less code, fine for simple ones.'

SQL Server (13)

Q44Query to find the 2nd highest salary
How to answerGive the DENSE_RANK version as your primary answer and say why: 'I use DENSE_RANK because it handles ties correctly, unlike a simple MAX-less-than.' Then mention the OFFSET alternative.
-- Handles ties:
SELECT Salary FROM (
  SELECT Salary, DENSE_RANK() OVER (ORDER BY Salary DESC) AS rnk
  FROM Employees
) t WHERE rnk = 2;

-- Simple alternative:
SELECT DISTINCT Salary FROM Employees
ORDER BY Salary DESC OFFSET 1 ROWS FETCH NEXT 1 ROWS ONLY;
Q45Clustered vs Non-Clustered index
How to answer'The clustered index IS the table — it defines physical row order, so one per table. Non-clustered is a separate structure pointing back to rows; I can INCLUDE columns to cover a query and avoid key lookups.'
CREATE CLUSTERED INDEX IX_Orders_Id ON Orders(Id);          -- physical order
CREATE NONCLUSTERED INDEX IX_Orders_Cust ON Orders(CustomerId)
    INCLUDE (Total, Status);   -- covering index — no key lookup needed
Q46CTE vs Temp Table
How to answer'A CTE is inline, single-statement, not materialized — great for readability and recursion. A temp table is materialized in tempdb with stats and indexes — better when I reuse a large intermediate result across statements.'
WITH Ranked AS (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY CustomerId ORDER BY Date DESC) rn
  FROM Orders
)
SELECT * FROM Ranked WHERE rn = 1;   -- latest order per customer
Q47Temp Table vs Table Variable
How to answer'Temp tables have statistics and support indexes, so the optimizer plans them well for large sets. Table variables have no stats — historically estimated as one row — so I keep them for small counts.'
Q48What causes deadlocks? / blocking
How to answer'Blocking is one session waiting on another's lock. A deadlock is a cycle — each holds what the other needs — and SQL Server kills one as the victim, error 1205. Usual causes: inconsistent object access order and long transactions. Fixes: consistent order, short transactions, right indexes, RCSI, and retry logic.'
-- Reduce reader/writer blocking with row versioning:
ALTER DATABASE MyDb SET READ_COMMITTED_SNAPSHOT ON;
Q49Parameter sniffing
How to answer'SQL Server caches a plan built for the FIRST parameter value; with skewed data that plan is terrible for other values. I fix it with RECOMPILE, OPTIMIZE FOR, or local variables — and 2022 has Parameter Sensitive Plans.'
SELECT * FROM Orders WHERE CustomerId = @cid
OPTION (RECOMPILE);   -- fresh plan per execution for skewed data
Q50Execution plan analysis
How to answer'I read the ACTUAL plan and hunt for scans where I expected seeks, key lookups I can eliminate with INCLUDE, big gaps between estimated and actual rows meaning stale stats, and spills to tempdb.'
SET STATISTICS IO, TIME ON;   -- measure logical reads + CPU alongside the plan
Q51Normalization vs Denormalization
How to answer'Normalize for integrity and write efficiency — OLTP is usually 3NF. Denormalize deliberately for read speed — reporting. When I need both, I use indexed views or a separate read model.'
Q52Transactions & isolation levels
How to answerOrder them by strictness and name what each prevents: 'Read Uncommitted allows dirty reads; Read Committed is the default; Repeatable Read stops non-repeatable reads; Serializable stops phantoms; Snapshot uses row versioning so readers don't block writers.'
SET TRANSACTION ISOLATION LEVEL SNAPSHOT;
BEGIN TRAN;
  -- readers see a consistent version, writers don't block them
COMMIT;
Q53Detect / find and remove duplicate records
How to answer'I find duplicates with GROUP BY … HAVING COUNT(*) > 1, then delete extras with ROW_NUMBER keeping one, and add a unique constraint so it can't recur.'
WITH d AS (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY Email ORDER BY Id) rn
  FROM Users
)
DELETE FROM d WHERE rn > 1;
Q54HAVING vs WHERE
How to answer'WHERE filters rows before grouping; HAVING filters groups after aggregation and can reference aggregates. I push as much as possible into WHERE for performance.'
SELECT CustomerId, COUNT(*) AS Orders
FROM Orders
WHERE Status = 'Complete'        -- before grouping
GROUP BY CustomerId
HAVING COUNT(*) > 5;             -- after aggregation
Q55How would you optimize a slow SQL query?
How to answer'I read the actual plan, make sure predicates are SARGable — no functions on the indexed column — add covering indexes, select only needed columns, update stats, and check for parameter sniffing.'
-- NON-SARGable — can't seek the index:
WHERE YEAR(OrderDate) = 2026
-- SARGable — can seek:
WHERE OrderDate >= '2026-01-01' AND OrderDate < '2027-01-01'
Q56How would you handle slow-running stored procedures?
How to answer'Same plan analysis, plus proc-specific fixes: SET NOCOUNT ON, replace cursors with set-based logic, handle parameter sniffing, and avoid the catch-all @p IS NULL OR col=@p pattern which kills index usage.'
-- catch-all params: use RECOMPILE or dynamic SQL instead of:
WHERE (@status IS NULL OR Status = @status)   -- forces a scan

Q57Azure App Service vs Functions / AKS / Kubernetes
How to answerFrame it as a control-vs-ops spectrum: 'App Service is PaaS with minimal ops; Functions is serverless and event-driven, scales to zero; AKS gives full control at the highest ops cost. I pick by how much control I need versus how much ops I want to own.'
Q58Azure Key Vault - use cases
How to answer'Central store for secrets, keys and certificates. The point is apps reach it with a Managed Identity, so there are zero credentials in code or config, plus rotation and audit logging.'
// App Service setting references the vault directly:
// @Microsoft.KeyVault(SecretUri=https://myvault.vault.azure.net/secrets/DbConn/)
// resolved at runtime via the app's managed identity — no secret in config.
Q59Azure Service Bus
How to answer'Enterprise broker — queues for point-to-point, topics for pub-sub. What makes it enterprise: sessions for ordering, dead-letter queues, duplicate detection, scheduled delivery. For streaming I'd use Event Hubs instead.'
Q60Azure Blob Storage vs Azure File Storage
How to answer'Blob is object storage for unstructured data with Hot/Cool/Archive tiers. Files is a managed SMB share you mount like a network drive — for lift-and-shift apps that expect a filesystem.'
Q61Azure API Management
How to answer'A managed gateway: publish, secure, rate-limit, transform, cache and version APIs with a developer portal — all via inbound/outbound policies without touching backend code.'
Q62Application Insights - how to monitor production apps?
How to answer'It's the APM layer — automatic request and dependency telemetry, exceptions, live metrics, distributed tracing via operation IDs. I query with KQL, build dashboards, and set alerts, sampling to control cost.'

requests
| where timestamp > ago(1h)
| summarize p95=percentile(duration,95) by name
| order by p95 desc
Q63CI/CD pipeline using Azure DevOps
How to answer'A YAML pipeline in the repo: CI builds, tests and publishes an artifact on PR; CD promotes that same artifact through dev, QA and prod with approvals, deploying infra via Bicep then swapping slots. Secrets come from variable groups linked to Key Vault.'
stages:
- stage: Build

- stage: Prod
  condition: succeeded()

Q64Docker vs Kubernetes
How to answer'Docker packages and runs one container — the box. Kubernetes orchestrates many across a cluster: scheduling, autoscaling, self-healing, rolling updates — the fleet manager.'
Q65Blue-Green vs Canary deployment
How to answer'Blue-green keeps two environments and switches all traffic at once — instant rollback by switching back. Canary sends a small percentage to the new version, watches metrics, then ramps up — safer detection, slower rollout.'

How to answerThe key insight interviewers want is the database: 'Slot-swap or rolling updates handle the app, but the real trick is backward-compatible migrations — expand-contract — so old and new code run against the same schema. Feature flags decouple release from deploy.'

Q67Infrastructure as Code (ARM / Bicep)
How to answer'I declare infrastructure in source-controlled Bicep for repeatable, reviewable environments with no drift. Bicep is the concise DSL that compiles to ARM, with modules and per-environment parameters, deployed from the pipeline.'

Q68Logging & distributed tracing
How to answer'Structured logging with a correlation ID on every request, and I propagate W3C trace context across service calls so one operation ID stitches the whole flow. Logs, metrics and traces are the three pillars, exported via OpenTelemetry.'

Q70Monolith vs Microservices
How to answerGive the balanced answer and the recommendation: 'Monolith is simpler to build and deploy but hits scaling and team-autonomy limits. Microservices give independent scaling and releases at the cost of distributed complexity. I'd start with a modular monolith and split when team size and scale justify it.'
Q71API Gateway - benefits and use cases
How to answer'A single entry point in front of services handling routing, auth, rate limiting, TLS termination and aggregation — it hides internal topology and centralizes cross-cutting policy. Azure APIM, Ocelot, YARP.'
Q72Circuit Breaker pattern
How to answerExplain the three states — that's the substance: 'After N failures it opens and fails fast; after a cooldown it goes half-open to test with one call; if that succeeds it closes. It stops me hammering a struggling dependency and prevents cascading failure.'
var pipeline = new ResiliencePipelineBuilder()
    .AddCircuitBreaker(new() { FailureRatio = 0.5, BreakDuration = TimeSpan.FromSeconds(30) })
    .Build();   // Polly / Microsoft.Extensions.Resilience
Q73CQRS pattern
How to answer'Separate the write model — normalized, consistent — from the read model — denormalized, fast. Each scales independently. The cost is eventual consistency between them, so I only use it where read and write shapes really differ.'
Q74Event-Driven architecture
How to answer'Services publish events; consumers react asynchronously — loose coupling and resilience. The challenges are eventual consistency and duplicate delivery, so consumers must be idempotent and I use the outbox pattern for reliable publishing.'
Q75Service-to-service communication / discovery
How to answer'Sync via HTTP or gRPC — fast but couples availability. Async via a broker — decoupled and resilient. Discovery is usually Kubernetes DNS or the cloud load balancer. I prefer async where the workflow allows.'
Q76Distributed transactions
How to answerLead with what NOT to do: 'I avoid two-phase commit — it doesn't scale. Instead a Saga: a chain of local transactions with compensating actions on failure, plus the outbox pattern so the DB write and event publish are atomic.'

// If payment fails → compensate: Release inventory (undo previous step)
Q77How to prevent cascading failures?
How to answer'Timeouts on every remote call, retries with exponential backoff and jitter for transient errors only, circuit breakers, bulkheads to isolate resource pools, and fallbacks for graceful degradation.'

How to answer'Redundancy across availability zones, stateless services behind load balancers with health probes, replicated data with automated failover, client-side retries and circuit breakers, and IaC so I can rebuild fast. I verify it with chaos testing.'
Q79How to handle high throughput?

How to answerQuantify it first — that impresses: '99.99% is about 52 minutes of downtime a year, so I need redundancy everywhere — multi-zone or multi-region, stateless services behind LBs, replicated data with automated failover, zero-downtime deploys, and tested DR runbooks.'

Q91How to handle 1 Million API requests per day?
How to answerDefuse the scary number with math — it shows judgment: 'A million a day is only about 12 requests a second on average — a well-built single service handles that. I'd still add a load balancer with two or three instances for HA, Redis for hot reads, and a queue for heavy work.'
Q92Duplicate financial transactions from retries — prevent?
How to answer'Idempotency keys: the client sends a unique key per logical transaction, I store the key with its result and replay it on retries. A unique constraint on the business key is the last line of defense, and consumers must be idempotent since messages redeliver.'
Q93One microservice keeps failing — stop cascading failures?
How to answer'I wrap calls to it with a timeout and circuit breaker so callers fail fast, serve a fallback where acceptable, and bulkhead its connection pool so it can't starve shared resources — while I fix the root cause.'

How to answer'Multi-region active-active, event-driven with outbox and idempotent consumers, a double-entry ledger, tokenized card data with PCI scope minimized, exactly-once via idempotency keys plus unique constraints, end-to-end tracing, and daily reconciliation.'

Q96How to ensure data consistency in distributed systems?
How to answer'I accept eventual consistency between services and keep local ACID transactions within one. Sagas with compensation for multi-service workflows, outbox/inbox for reliable messaging, idempotent consumers, optimistic concurrency, and reconciliation jobs to catch drift.'
Q97How to handle partial failures in microservices?
How to answer'I assume any call can fail: timeouts, retries with backoff for transient errors only, circuit breakers, fallbacks, bulkheads, dead-letter queues for poison messages, and idempotent operations so retries are safe.'
Q98A new feature causes performance degradation — identify and fix?
How to answer'If it's flagged I turn it off for instant relief, then compare traces before and after to isolate the slow span. Usual suspects: an N+1 query, a missing index for the new query, or sync-over-async. Fix, load test, re-enable via canary.'
Q99How to handle rollback if deployment fails?
How to answer'I automate it — post-deploy health checks gate the pipeline and probe failures trigger auto-rollback to the previous artifact. Releases are immutable and versioned, migrations are expand-contract, and feature flags are the fastest rollback of all.'

Coding & DSA (13)

Q120Reverse a string
How to answerState the approach and complexity before you code — that's the habit interviewers grade: 'Two pointers swapping inward, O(n) time and O(1) extra.'
public static string Reverse(string s) {
    var a = s.ToCharArray();
    int i = 0, j = a.Length - 1;
    while (i < j) { (a[i], a[j]) = (a[j], a[i]); i++; j--; }
    return new string(a);
}
// Library one-liner: new string(s.Reverse().ToArray());
Q121Check if a string is a palindrome
How to answer'Two pointers from both ends, O(n)/O(1). I'd clarify whether to ignore case and punctuation first.'
public static bool IsPalindrome(string s) {
    int i = 0, j = s.Length - 1;
    while (i < j) if (s[i++] != s[j--]) return false;
    return true;
}

How to answerOffer both, and note why XOR is safer: 'Sum formula is cleanest; XOR avoids any overflow risk. Both O(n)/O(1).'

Q123Remove duplicate characters
How to answer'One pass with a HashSet of seen chars, appending only new ones — preserves first-occurrence order, O(n).'
public static string RemoveDupes(string s) {
    var seen = new HashSet<char>(); var sb = new StringBuilder();
    foreach (var c in s) if (seen.Add(c)) sb.Append(c);

Q124Find first non-repeating character
How to answer'Two passes: count occurrences, then return the first char with count one. O(n), O(1) for a bounded alphabet.'
public static char? FirstUnique(string s) {
    var count = new Dictionary<char,int>();
    foreach (var c in s) count[c] = count.GetValueOrDefault(c) + 1;
    foreach (var c in s) if (count[c] == 1) return c;
    return null;
}
Q125Implement an LRU Cache
How to answer'Dictionary plus a doubly linked list — Get moves the node to the front, Put inserts at front and evicts the tail when full. Both O(1).'
public class LruCache<K,V> {
    private readonly int _cap;
    private readonly Dictionary<K, LinkedListNode<(K,V)>> _map = new();
    private readonly LinkedList<(K,V)> _list = new();
    public LruCache(int cap) => _cap = cap;
    public V? Get(K k) {
        if (!_map.TryGetValue(k, out var n)) return default;
        _list.Remove(n); _list.AddFirst(n); return n.Value.Item2;
    }
    public void Put(K k, V v) {
        if (_map.TryGetValue(k, out var n)) { _list.Remove(n); }
        else if (_map.Count >= _cap) {
            _map.Remove(_list.Last!.Value.Item1); _list.RemoveLast();
        }
        var node = new LinkedListNode<(K,V)>((k, v));
        _list.AddFirst(node); _map[k] = node;
    }
}
Q126Implement Binary Search
How to answer'Sorted array required. Note the mid = lo + (hi-lo)/2 trick to avoid overflow. O(log n).'
public static int BinarySearch(int[] a, int target) {
    int lo = 0, hi = a.Length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (a[mid] == target) return mid;
        if (a[mid] < target) lo = mid + 1; else hi = mid - 1;
    }
    return -1;
}

Q131Frequency of digits in a number (0–9 counts)
How to answer'A size-10 int array, extract each digit with mod-10 and divide-by-10. Handle zero and negatives. O(number of digits).'
static int[] DigitFrequency(int n) {
    var count = new int[10];
    n = Math.Abs(n);
    if (n == 0) { count[0] = 1; return count; }
    while (n > 0) { count[n % 10]++; n /= 10; }
    return count;
}
Q132Largest and smallest digit in a number
How to answer'Same extraction loop, tracking max and min. Start max at 0 and min at 9, handle negatives with Abs. O(number of digits).'
static (int min, int max) MinMaxDigit(int n) {
    n = Math.Abs(n);
    int min = 9, max = 0;
    if (n == 0) return (0, 0);
    while (n > 0) { int d = n % 10; max = Math.Max(max, d); min = Math.Min(min, d); n /= 10; }
    return (min, max);
}
