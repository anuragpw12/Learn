What is Angular?

- automatic DOM updates (change detection)

- built-in tools for forms, HTTP, routing

Data Binding
	- Data Binding is the mechanism that connects the component (TypeScript) and the template (HTML).
	- maunal dom work problem solve $('#name').text(user.name);

- 4 Types - 
	- 1. Interpolation (One-way: TS → HTML)
		HTML = <h1>{{ username }}</h1>
		TS = username = "John";
		
	- 2. Property Binding  (One-way: TS → HTML)
		TS - isDisabled = true;
		HTML - <button [disabled]="isDisabled">Save</button>

	- 3. Event Binding 
		html - <button (click)="login()">
		TS - login() {
		  console.log("Login clicked");
		}
		
	- 4. Two-Way Binding 
		- TS - username = '';
		HTML - 
		<input [(ngModel)]="username">
		<p>{{username}}</p>

Lifecycle hooks
	- Lifecycle hooks are special methods that allow us to execute code at specific stages.

1. constructor() → Runs when the class is instantiated. Only use it for DI
2. ngOnChange() → ngOnChanges fires only if @Input() is present, triggered every time the input reference changes. 
				  On first load it fires before ngOnInit, 
				  after that it fires independently whenever parent passes a new value.
3. ngOnInit() → Runs once after the component is initialized.; ideal for HTTP calls
4. ngDoCheck() → Called on every change detection cycle.
5. ngAfterContentInit() → Fires once after <ng-content> is projected into the component.
6. ngAfterContentChecked() → ngAfterContentChecked fires every time Angular checks the projected content 
                             (i.e. after every change detection cycle on the <ng-content>)
7. ngAfterViewInit() → Called once after the component's own view and all child views are initialized.
                       Good place to interact with @ViewChild
8. ngAfterViewChecked() → Called after every check(Change Detection) of the component's view. 
                          Don't trigger new changes here — it causes infinite loops.
9. ngOnDestroy() → Called just before the component is removed from the DOM. Always clean up here.



| Feature        | `@Input()`                         | `ng-content`                                       |
| -------------- | ---------------------------------- | -------------------------------------------------- |
| What is passed | Data / value                       | HTML / template                                    |
| Hook to use    | `ngOnChanges`                      | `ngAfterContentInit`                               |
| Example        | `[title]="'Hello'"`                | `<app-card><h2>Hello</h2></app-card>`              |
| Access via     | Property binding                   | `@ContentChild`                                    |
| When available | `ngOnChanges` / `ngOnInit`         | `ngAfterContentInit`                               |
| Use case       | Pass string, number, object, array | Pass reusable HTML blocks like modals, tabs, cards |
| Direction      | Parent → Child (data)              | Parent → Child (template)                          |

16.How do you prevent unnecessary API calls?
	1. Debounce Input

	Use debounceTime

	Delay API call until user stops typing

	🔷 2. Avoid Duplicate Calls

	Use distinctUntilChanged

	Prevent same value requests

	🔷 3. Cancel Previous Requests

	Use switchMap

	Keeps only latest API call

	🔷 4. Implement Caching

	Store API response in service

	Return cached data instead of calling again

	🔷 5. Share API Response

	Use shareReplay(1)

	Avoid multiple API calls for multiple subscribers

	🔷 6. Avoid Multiple Subscriptions

	Don’t subscribe multiple times to same API

	🔷 7. Conditional API Calls

	Call API only if data is not already available

	🔷 8. Use Guards / Resolvers

	Fetch data once before route loads

	🔷 9. Throttle Frequent Events

	Use throttleTime (scroll, clicks)

17. How to implement caching in Angular?

	🔷 1. Service-Level Caching

	Store API response in a service variable

	Return cached data if available

	Avoid repeated API calls

	🔷 2. Use shareReplay(1)

	Share single API response across subscribers

	Prevent multiple HTTP calls

	🔷 3. Cache with Expiry (TTL)

	Store timestamp with data

	Refresh data after specific time

	🔷 4. HTTP Interceptor Caching

	Intercept GET requests

	Cache responses globally

	🔷 5. Browser Storage

	Use localStorage / sessionStorage

	Persist data across page reload

	🔷 6. NgRx Store (Large Apps)

	Store API data in global state

	Reuse across components

18. How to prevent memory leaks
	
	🔷 1. Unsubscribe Observables

	Use unsubscribe() in ngOnDestroy

	🔷 2. Use takeUntil

	Preferred for multiple subscriptions

	Auto unsubscribe on destroy

	🔷 3. Use async Pipe

	Automatically handles subscribe/unsubscribe

	🔷 4. Avoid Nested Subscriptions

	Use switchMap, mergeMap instead

	🔷 5. Clean Event Listeners

	Remove addEventListener in ngOnDestroy

	🔷 6. Use take(1) / first()

	Auto complete after one emission

	🔷 7. Avoid Long-Lived Streams

	Be careful with interval, fromEvent

	🔷 8. Complete Subjects

	Call complete() on Subjects

	🔷 9. Use Angular Built-in Handling

	HTTP calls auto complete (no need to unsubscribe)



19.How to secure Angular routes?
	
	🔷 1. Use Route Guards

CanActivate → allow/deny access

CanActivateChild → child routes

CanLoad → restrict lazy modules

CanDeactivate → prevent leaving page

🔷 2. Authentication Check

Verify user login (JWT/token)

Allow only authenticated users

🔷 3. Redirect Unauthorized Users

Navigate to /login if not authorized

🔷 4. Role-Based Authorization

Restrict routes based on user roles (Admin/User)

🔷 5. Secure Lazy Loading

Use CanLoad to block module access

🔷 6. Use HTTP Interceptor

Attach JWT token to API requests

Handle auth errors (401/403)

🔷 7. Secure Token Storage

Store token safely (avoid exposure)

🔷 8. Backend Validation (Very Important)

Always validate on server

Frontend alone is not secure


20.How to implement Role-Based Access Control (RBAC)?

🔷 1. Store User Role

Get role from backend (JWT / API)

Store in service or token

🔷 2. Auth Service

Manage user authentication and roles

Provide method to get user role

🔷 3. Route Guard (CanActivate)

Check user role before route access

Compare with expected role

🔷 4. Route Configuration

Define roles in route data

🔷 5. Redirect Unauthorized Users

Navigate to login / access denied page

🔷 6. UI-Level Access Control

Show/Hide elements using role (*ngIf)

🔷 7. HTTP Interceptor (Optional)

Attach token/role to API requests

🔷 8. Backend Validation (Very Important)

Validate roles on server

Frontend alone is not secure
